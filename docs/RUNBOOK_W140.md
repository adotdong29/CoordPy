## 3. Parser-neutrality + no-leakage + no-confound rule — LOCKED

- **Parser-neutral I/O (HC1):** every minted task uses the canonical one-logical-item-per-line normal form (`coordpy.parser_neutral_io_v1`); REJECTED unless a strict per-line reader and a read-all-tokens reader recover byte-identical structured data for every minted+secret case (the W136 confound is machine-impossible to recur). Unchanged from W139/W138/W137/W132.
- **No-leakage of the FIELD:** the model under test sees ONLY the `statement` + PUBLIC `samples`. NEVER model-facing: `ref_source`/`naive_source`/`brute_source`, the graded `secret_cases`, any hidden I/O. Grading is `grade_on_secret_v1` on the DISJOINT hidden bank ⇒ tests GENERALISATION.
- **No-leakage of the TUTOR (the W140-specific integrity contract).** A tutor is FAMILY-LEVEL or TRAIN-split-derived, never the target answer. The deterministic `tutor_leak_gate_v1` MUST pass before a tutor is emitted to any model; it asserts ALL of:
  1. **`reference_overlap`** — the tutor's code-bearing fields (skeletons, key-moves) do NOT reproduce the family's `ref_source` beyond a normalized longest-common-substring / token-overlap threshold (no reference paste under the guise of pseudocode).
  2. **`public_only_literals`** — every numeric / array literal that appears in any worked trace or example byte-equals a declared PUBLIC sample input, and intersects NO graded `secret_cases` input (no worked example equal to a hidden case).
  3. **`no_closed_form_answer`** — the tutor contains a PROCEDURE (control flow), not a terminal expression that, evaluated on the public sample, yields the public OUTPUT (no answer-formula leak).
  4. **`hole_substance`** (for any skeleton) — holes are non-trivial semantic decisions; the skeleton with holes naively stubbed (`pass`/`0`/`True`) FAILS the public sample (so a "skeleton" that's the solution with cosmetic holes is rejected).
  5. **`one_liner_family_guard`** — if the family's reference is below a triviality threshold (≤1 statement, no branching) the name-card is DISABLED for that family (a name == the code for one-liners). Both W140 shared families (`count_pairs_sum_le_t`, `subarrays_sum_and_range`) are certified MULTI-STEP ⇒ cards permitted (the gate certifies this per family, never assumes it).
  6. **`discriminator_shape_only`** — "fails on inputs like…" descriptions are SHAPE words only ("a few large outliers among small values"), never a concrete array; any literal present is run naive-vs-ref and rejected if they disagree (it is itself a discriminating/hidden-like case).
  7. **`train_test_disjoint`** — every TRAIN instance a tutor is compiled from has empty input-hash intersection with the graded `secret_cases` of every eval/frontier instance (no train↔test collision; TRAIN seed base `140_1×10^5` is byte-disjoint from dev/eval/frontier `140_2/3/4×10^5`).
- **No-confound / anti-cheat:** no official-task paraphrases; no accepted-solution reuse; no secret-case reuse; no per-instance tutor that is effectively the target solution; no formatting artifact that explains failures (HC1 enforces). The leakage guard reuses the **corrected contiguous-block tripwire** lineage (W127's fix — never the per-line check that false-positived on boilerplate like `n,k=map(int,input().split())`).
- **Same-budget fairness:** the tutor is COMPILED at build/calibration time (TC1 deterministic from the owned oracle = $0 NIM; TC2 from TRAIN transcripts = NIM paid ONCE per family at calibration, OUTSIDE the graded K budget — exactly the W139 witness-usability accounting). At bench time the tutor is STATIC PROMPT TEXT injected into the same K attempts — **zero extra model calls**; every arm is exactly K=5 calls like A1/B0. The tutor is input-side; the K (output) budget is identical across arms. No silent call expansion.

- **TC2 — Witness-to-Rewrite Template.** A compact mapping `observed_witness → named_root_cause → specific_rewrite` (e.g. `TLE@N → "O(N²) double loop" → TC1-skeleton`; `WA-passes-public → "dropped the (max−min)≤R constraint" → "add min/max deques + OR-shrink"`), with a substantive-holes skeleton as the rewrite target. - **TC3 — Compressed Stress Tutor (smallest sufficient).** The minimal object that still lifts the weak tier — one budget fact + one stress/technique line + one rewrite/bug rule, under a hard token budget. Greedily reduced from TC1. Defuses the 8B long-text-instruction-breakdown + revert-to-naive failure modes.
- **TC4 — Keep/Apply/Abstain Tutor Controller (the LEAD).** The W139 capability-matched controller, with the APPLY branch consuming a TUTOR object (TC1/TC2/TC3) instead of the raw per-problem witness, routed by the per-tier `tutor_usability_rate`: tutor-ineligible tier → KEEP (all K plain, `T4 ≡ A1`, non-negativity by construction); eligible tier → attempt-0 plain, attempt-1 tutor-reflexion, k≥2 APPLY iff the tutored draft did not strictly worsen the public-sample pass count else per-problem REVERT to plain. - **TC5 — Weak-Tier Staged Fallback (conditional; only if Lane α honestly warrants).** Progressive disclosure gated by execution feedback: Stage0 TC3 → Stage1 +TC1 key-move+primitive → Stage2 +TC2 holes-skeleton → Stage3 +invariants/bug-warnings/public-sample micro-trace/self-check (never a full solution). Escalate only on failed execution at the current stage; stop at first pass. Emitted leak budget = highest stage reached. Fielded only if TC1–TC4 show the weak tier is dose-sensitive (else recorded as not-warranted).
- **T6 — Negative-control simple tutor (fake-different probe).** A deliberately content-free "tutor" (a generic do-better instruction with an empty technique label); the structural fake-different test (`fake_different_report_v1` / `tutor_is_genuinely_new_v1`) MUST classify it FAKE_DIFFERENT, proving the bench rewards real compiled teaching, not prompt decoration.
