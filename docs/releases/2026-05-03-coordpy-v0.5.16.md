# CoordPy 0.5.16

First public CoordPy release and repo-wide rename from Wevra to CoordPy.
This release makes `coordpy` the shipped package name, `vision_mvp.coordpy`
the stable Python import surface, and `coordpy` / `coordpy-import` /
`coordpy-ci` / `coordpy-capsule` the public CLIs. The rename carries through
package metadata, docs, tests, workflows, and result artifacts; the runtime
contract itself remains the same narrow audited capsule contract.

## Stable vs experimental

Stable in this release:

- `vision_mvp.coordpy` SDK: `RunSpec`, `run`, `RunReport`, `SweepSpec`,
  `run_sweep`, `CoordPyConfig`, profiles, CI/import helpers, extensions, and
  capsule primitives.
- Public CLIs: `coordpy`, `coordpy-import`, `coordpy-ci`, `coordpy-capsule`.
- On-disk contracts: `coordpy.capsule_view.v1`,
  `coordpy.provenance.v1`, and `phase45.product_report.v2`.

Experimental but included:

- `vision_mvp.coordpy.__experimental__`.
- The W22..W42 trust-adjudication / multi-agent coordination ladder.
- The R-69..R-89 benchmark drivers and bounded live cross-host probes.

## Install

Install from a clone with `pip install -e .`, or from PyPI with
`pip install coordpy` once published. `pipx install coordpy` is suitable when
you only want the CLIs. NumPy is the only required dependency; optional extras
include `coordpy[scientific]`, `coordpy[dl]`, `coordpy[heavy]`,
`coordpy[crypto]`, `coordpy[docker]`, and `coordpy[dev]`.

## Out of scope

This release does not claim:

- transformer-internal trust transfer (`W42-C-NATIVE-LATENT`);
- K+1-host disjoint topology beyond the current two-host setup
  (`W42-C-MULTI-HOST`);
- stability for the research surface under `__experimental__`.

## Migration from Wevra

- Python imports: `vision_mvp.wevra` -> `vision_mvp.coordpy`
- Package / CLI name: `wevra` -> `coordpy`

Update downstream imports and command invocations to the CoordPy names. Treat
the CoordPy stable surface as the supported contract; anything under
`vision_mvp.coordpy.__experimental__` remains non-stable research API.
