## 3. Parser-neutrality + no-leakage rule — LOCKED (unchanged from W138/W137/W132)

- **Parser-neutral I/O (HC1):** every minted task uses the canonical one-logical-item-per-line normal form (`coordpy.parser_neutral_io_v1`); REJECTED unless a strict per-line reader and a read-all-tokens reader recover byte-identical structured data for every minted+secret case. - **No-leakage:** the model under test sees ONLY the `statement` + PUBLIC `samples`. NEVER model-facing: `ref_source`/`naive_source`/`brute_source`, the graded `secret_cases`, any hidden I/O. Grading is `grade_on_secret_v1` on the DISJOINT hidden bank ⇒ tests GENERALISATION. The witness/feedback is reconstructible only from (a) the candidate's own program, (b) the public statement+samples, (c) owned-oracle executions on FRESH witness-seed probes byte-disjoint from `secret_cases` — emitting only oracle OUTPUTS + derived structural summaries. **The large-probe counterexample witness (`Nb`) keeps the SAME disjointness guard** (`leakage_clean`: the probe input is not byte-equal to any graded secret-case input).
- **No-confound / anti-cheat:** no official-task paraphrases; no accepted-solution reuse; no secret-case reuse; no formatting artifact that explains failures (HC1 enforces); per-instance novelty within a template.
- **Capability-matched controller fairness:** the only NEW signal `Cm` consumes is (i) the per-tier `witness_usability_rate` measured at CALIBRATION (paid in calibration, not in the graded K budget), and (ii) per-attempt PUBLIC-sample pass counts of its OWN candidates ($0, public-only). ## 8. Same-budget mechanism arms + earn rule — LOCKED before dev NIM

All arms **K=5, same model, temp 0.7, `max_tokens=1536`, `executor_timeout_s=8.0`.** Attempt-0 = byte-identical standard prompt; K attempts; one model call per attempt; no early stop; graded on `secret_cases`; scored in the "B" slot so `arm − A1 ≡ B − A1` via the verbatim W108 `_evaluate_phase2_gates`/`_mlb_rates`. Witness generation (owned-oracle on disjoint probes) is **$0 NIM, paid OUTSIDE the K budget**.

- **A0** — single shot, pass@1.
- **A1** — K=5 self-consistency, pass@K (the headroom baseline the lead must beat).
- **B0** — W120/W132 blind reflexion (judge-reject bit + stderr + public-sample results). The second baseline.
- **C0** — exact-oracle COMPLEXITY witness (`exact_oracle_witness_v1` ARM_C2), the W138 anchor winner — diagnostic (shows the BLIND-apply ceiling, incl. the 8B −25 it caused).
- **Cm** — **capability-matched controller (the W139 LEAD).** Same-budget K. Routing: (1) if the tier is NOT witness-eligible (`witness_usability_rate < τ_wu`) → all K attempts are PLAIN self-consistency draws ⇒ `Cm ≡ A1` (KEEP — never hurts the weak tier); (2) if witness-eligible → attempt-0 standard, attempt-1 witness-reflexion (the per-problem probe), then for k≥2 APPLY the witness iff the probe did NOT strictly worsen the public-sample pass count, else REVERT this problem to plain self-consistency. Consumes: the per-tier capability prior (§6) + per-attempt public-pass counts of its own candidates. *Hypothesis: routing by measured capability makes the witness gain non-negative on weak tiers (R5/R2 — small models need a verifier matched to their capability; arXiv:2404.17140) while preserving the anchor gain.*
- **Nb** — **large-probe COUNTEREXAMPLE witness (2nd-mode revival).** `run_witness_arm_v1` arm C1 fed a probe set whose `.small` carries LARGE structured inputs (token cap raised to `LARGE_PROBE_TOKEN_CAP = 6000`), so a counterexample that only appears on large inputs (the HIDDEN_EDGE families) is now found. *Hypothesis: the W138 N0 +0 was a probe-size firing artifact, not a capability limit; a large-probe counterexample revives the 2nd mode ⇒ multi-MODE span.*
- **X2** — deliberately simple NEGATIVE CONTROL (relabeled reflexion) the structural fake-different test must classify `FAKE_DIFFERENT`. Scored at $0 (≡ B0).
