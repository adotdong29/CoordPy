# Frontier-relevance audit — W126 (family-adapted repair synthesis)

Classifies the W126 additions and re-affirms the standing arsenal/baseline/dead/anti-pattern
columns. Supplements the W125/W123/W121 audits; all prior classifications remain in force.

## Active-frontier arsenal (NEW in W126)

* **`coordpy.family_adapted_repair_synthesis_v1`** — the new-trajectory synthesis slate:
  S1 cross-candidate AST splice, S2 digest-grounded repair, S3 exposed-family motif harden,
  **S-CONS trust-weighted output-consensus dispatcher**. The FIRST module to wire the
  repair/consensus arsenal (`adversarial_consensus_repair_v1.trust_weighted_consensus_v1`,
  `executor_grounded_patcher_v1` digest) onto the official-ICPC code path
  (graphify-confirmed: previously 4 hops apart through a trivial `str` node — now a 1-hop
  `imports_from` edge). **Active frontier**: this is the synthesis instrument a future
  OPERATOR-GREENLIT fresh pilot or a code-COMPETENT local model would drive; S-CONS is a
  genuinely new program shape (per-case trust-weighted plurality) that W125's selection
  could not express.
* **`SynthesisLeakageGuardV1` (provenance-aware)** — a reusable no-leakage harness: flags a
  secret/answer byte-run only when it is ABSENT from the provenance (source generations +
  samples), distinguishing a real injection (an accidental secret-file read) from a
  base-model coincidence (a generation literal that happens to equal a short secret answer,
  e.g. emoticons `^_^`). Active frontier (the correct discipline for any future synthesis
  that recombines model output near secret data).
* **`_run_capped_v1`** — a hardened candidate executor (bounded stdout + process-group kill)
  that runs pathological synthesized code (infinite `print` loops) safely as a fast screen,
  with the official `grade_on_secret_v1` as authority. Active frontier (reusable for any
  synthesis that executes machine-generated programs).
* **The $0 synthesis precursor** (`synthesize_and_measure_problem_v1` + `run_w126_synthesis_precursor_v1`)
  — re-route AND re-synthesize over already-paid generations, grade with the official oracle,
  at $0, to decide whether NEW trajectories justify a fresh pilot. Active frontier (the
  cheapest honest "is new-trajectory synthesis worth buying?" precursor; the synthesis-side
  sibling of the W125 re-routing headroom replay).

## Anti-patterns (REMAIN explicit anti-patterns; W126 reinforces)

Bounded-context / compaction / generic summarization / "cram less, truncate better" remain
anti-patterns, NOT the frontier path. W126 explicitly did NOT drift into them — it built a
real new-trajectory synthesis slate (splice/repair/harden/consensus) and proved sharply that
$0 synthesis is dead on capability failures. Reranking-only and reflexion-relabeling remain
insufficient (W125); W126 adds that even genuine NEW-program synthesis is dead at $0 here.

## Do-not-claim (see `docs/HOW_NOT_TO_OVERSTATE.md` W126 section)

The synthesis slate is real and arsenal-native, but it created ZERO new resistant trajectories
at $0; the resistant field is synthesis-capped (oracle ceiling 0), not merely re-routing-capped;
a fresh pilot is an untested hope, deliberately not bought; the new-trajectory hope requires NEW
model capability (a stronger/code-competent model or an operator-greenlit NIM spend), not more
deterministic synthesis; W89+W105 stand as the only two retirements; multi-agent context is not
"solved".
